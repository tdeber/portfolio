# VotoTechAR – Proyecto de Ingeniería de Software

Proyecto académico desarrollado en la materia **Ingeniería de Software II**  
Carrera: Ingeniería Informática.

---

## Contexto del Proyecto

El proyecto consistió en la simulación del desarrollo de un sistema de software real a partir de los requerimientos de un cliente ficticio.

Se propuso diseñar **VotoTechAR**, una plataforma web orientada a automatizar y optimizar diferentes etapas del proceso electoral en Argentina.

El sistema contempla funcionalidades como:

- Registro online de votantes
- Validación de identidad mediante servicios externos
- Gestión de candidaturas políticas
- Generación automática del padrón electoral
- Publicación de resultados en tiempo real
- Generación de reportes administrativos

El trabajo se desarrolló siguiendo un enfoque metodológico propio de la Ingeniería de Software, abarcando análisis de requerimientos, modelado, diseño conceptual y definición de pruebas funcionales.

---

## Objetivos

El objetivo principal del proyecto fue aplicar conceptos y herramientas de Ingeniería de Software para:

- Comprender y analizar los requerimientos del sistema
- Modelar el comportamiento del sistema mediante UML
- Diseñar la arquitectura conceptual del dominio
- Definir escenarios y casos de prueba funcionales

---

## Etapas del Proyecto

### Modelado de Casos de Uso

-  Identificación de actores del sistema
- Construcción del diagrama general de casos de uso
- Redacción de fichas detalladas
- Modelado de relaciones <<include>> y <<extend>>

### Diseño del Sistema

- Definición del modelo de dominio
- Elaboración de diagrama de clases
- Identificación de roles y responsabilidades
- Diseño conceptual de la arquitectura del sistema

### Pruebas Funcionales

-  Derivación de escenarios de prueba a partir de casos de uso
- Definición de casos de prueba
- Validación de flujos normales y alternativos
